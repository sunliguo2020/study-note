在 VMware vSphere 环境中，将虚拟机在不同存储之间迁移有多种方法，具体取决于你的环境配置和业务需求。以下是几种常见方式：

---

## 一、通过 vSphere Client 进行存储迁移

### 1. **冷迁移（Cold Migration）**

虚拟机处于关机状态时迁移存储。

- 右键点击虚拟机 → **迁移** → 选择 **更改存储**
- 选择目标存储（数据存储或数据存储集群）
- 可选择更改虚拟机磁盘格式（厚置备/精简置备）
- 完成后启动虚拟机

**适用场景**：对停机时间无要求、简单直接的迁移

---

### 2. **存储 vMotion（Storage vMotion）**

虚拟机处于**开机状态**时实时迁移存储，**业务不中断**。

- 右键点击虚拟机 → **迁移** → 选择 **仅更改存储**
- 选择目标数据存储
- 可选择磁盘格式（厚置备懒置零、厚置备置零、精简置备）
- 点击完成，迁移过程中虚拟机保持运行

**前提条件**：

- vSphere 环境需具备 vMotion 许可
- 源和目标数据存储需同时被 ESXi 主机访问
- 虚拟机不能有快照（6.5 版本开始部分支持有快照的迁移，但有限制）

---

### 3. **跨主机 + 存储迁移（vMotion + Storage vMotion）**

同时更改计算资源和存储资源。

- 右键点击虚拟机 → **迁移**
- 选择 **更改计算资源和存储**
- 选择目标主机/集群和目标数据存储
- 虚拟机可保持开机状态，业务不中断

---

## 二、通过命令行进行迁移

### 使用 `vmkfstools` 复制 VMDK 文件

适用于需要精细控制或脚本化操作的场景：

```bash
# 复制虚拟磁盘
vmkfstools -i /vmfs/volumes/source_ds/vm/vm.vmdk /vmfs/volumes/target_ds/vm/vm.vmdk -d thin

# 注册虚拟机
vim-cmd solo/registervm /vmfs/volumes/target_ds/vm/vm.vmx
```

---

## 三、通过存储阵列层迁移

如果使用 **存储阵列级别的复制或快照技术**（如 VPLEX、SRM、存储厂商的异地复制功能）：

1. 在存储层完成数据同步
2. 在 vCenter 中执行 **存储重新扫描**
3. 从清单中移除原虚拟机
4. 在目标存储上注册虚拟机

这种方式通常用于灾备或大规模迁移场景。

---

## 四、迁移前注意事项


| 检查项           | 说明                                                        |
| ---------------- | ----------------------------------------------------------- |
| **存储兼容性**   | 确保目标数据存储支持虚拟机存储格式（VMFS/NFS/vSAN）         |
| **磁盘格式转换** | 迁移时可同时转换磁盘格式（如精简→厚置备）                  |
| **快照影响**     | Storage vMotion 支持有快照的虚拟机，但可能增加迁移时间      |
| **许可要求**     | Storage vMotion 需要 Enterprise Plus 许可                   |
| **资源占用**     | 迁移过程中会产生网络和存储 IO 负载，建议在业务低峰期进行    |
| **独立磁盘**     | 配置为独立磁盘（非持久）的磁盘无法通过 Storage vMotion 迁移 |

---

## 总结


| 迁移方式            | 是否需要关机 | 是否更改主机 | 许可要求        |
| ------------------- | ------------ | ------------ | --------------- |
| 冷迁移（更改存储）  | ✅ 需要      | ❌ 不变      | 基础版          |
| Storage vMotion     | ❌ 无需      | ❌ 不变      | Enterprise Plus |
| 全迁移（主机+存储） | ❌ 无需      | ✅ 可变更    | Enterprise Plus |
